# 最適化シミュレーター: 線形計画法 vs 二次計画法

線形計画法（Linear Programming, LP）と二次計画法（Quadratic Programming, QP）の違いを視覚的に理解するためのインタラクティブなシミュレーターです。

## アプリケーション概要

このアプリケーションは、最適化問題における線形目的関数と二次項（正則化項）を含む目的関数の違いを、リアルタイムで可視化します。

### 主な機能

- **インタラクティブなパラメータ調整**: スライダーで目的関数の係数（Cx, Cy）と正則化パラメータ（ε）を調整
- **リアルタイム最適化**: パラメータ変更に応じて、バックエンドで最適化計算を実行し、結果を即座に表示
- **視覚的な可視化**: 
  - 等高線図で目的関数の形状を表示
  - 制約領域（三角形）を表示
  - 最適解を赤い点で表示

### 目的関数

```
minimize: cx·x + cy·y + (ε/2)·(x² + y²)
```

- **ε = 0**: 線形計画法（LP）- 目的関数は線形
- **ε > 0**: 二次計画法（QP）- 二次項（正則化項）が追加され、中心に戻ろうとする力が働く

### 制約条件

```
0 ≤ x ≤ 1
0 ≤ y ≤ 1
x + y ≤ 1
```

制約領域は、原点(0,0)、点(1,0)、点(0,1)を結ぶ三角形です。

## セットアップと実行方法

### バックエンドのセットアップ

```bash
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

バックエンドは `http://localhost:8000` で起動します。

### フロントエンドのセットアップ

```bash
cd frontend
npm install
npm run dev
```

フロントエンドは `http://localhost:5173` で起動します（Viteのデフォルトポート）。

## 使い方

1. バックエンドとフロントエンドの両方を起動
2. ブラウザでフロントエンドのURLにアクセス
3. スライダーで以下のパラメータを調整：
   - **Cx**: X方向の線形項の係数（-2 ～ 2）
   - **Cy**: Y方向の線形項の係数（-2 ～ 2）
   - **ε (イプシロン)**: 二次項の係数（0 ～ 5）
4. グラフ上で最適解（赤い点）の位置と等高線の形状の変化を観察

### 観察のポイント

- **ε = 0（線形計画法）**: 最適解は制約領域の端（頂点）に位置することが多い
- **ε > 0（二次計画法）**: 二次項により、最適解が領域の中心寄りに移動する傾向がある
